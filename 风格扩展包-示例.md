## 一、段落结构：不是碎，是"呼吸"

### 1.1 段落长度配比（铁律）

| 段落类型 | 字数范围 | 占比 | 功能 |
|----------|----------|------|------|
| 长段 | 80-200字 | 约40% | **主力**。铺背景、做论证、摆数据、讲逻辑 |
| 中段 | 30-80字 | 约35% | 过渡、转场、展开一个小论点 |
| 短段 | 5-30字 | 约25% | **点缀**。情绪爆破、金句、吐槽、反转 |

**核心认知：长段才是主力，短段是情绪爆破点，不是常态。**

通篇短段 = 碎嘴子 = AI味。节奏像呼吸：吸气（长段铺信息）→ 屏息（中段转场）→ 呼气（短段释放情绪）。

### 1.2 段落过渡方式

**绝不用逻辑连接词**（首先/其次/然而/因此/此外/同时），用以下三种方式：

**① 情绪转场**：直接用语气变化切换。前面还在冷静分析，突然一句吐槽，段落就转了。
**② 设问转场**：用一个问题把读者拉到下一个话题。"看出问题没有？""更有可能的原因是什么呢？"
**③ 叙事跳切**：像电影剪辑一样直接跳到下一个场景，不做过渡交代。

### 1.3 开头信息密度（铁律）

**前三段必须密集释放**：互联网梗、案例、数据、反转——开头不是用来"引入话题"的，是用来"炸场"的。

**禁止干巴巴设问开头**（如"你有没有想过XXX？"）。这种开头信息密度为零，读者一句话都记不住。

**正确做法**：在前三段内完成——
1. 一个有辨识度的梗/名言/案例（迅速建立调性）
2. 一个跟主题相关的事实/数据/反转（锚定话题）
3. 一个极短的判断或情绪爆破（点题）

**示范**：
> 中国互联网历史上有句名言，领先半步是先驱，领先一步是先烈。Meta 在元宇宙远未成熟的时候选择了 All In，扎克伯格因此被人送上外号"北美李彦宏"。ChatGPT 在 LLM 技术开始产生质变之时推出，凭借强大的技术和产品顺势成为 AI 时代的执牛耳者。
>
> 可 2026 年了，御三家的模型能力已经没有代差……那你为什么还在用 ChatGPT 不换其他模型？
>
> 因为你懒。

三段话里塞进了：一句互联网名言 + 一个 Meta 反面梗 + ChatGPT 正面定位 + 时间锚点急转 + 三个字单句成段点题。这才叫开头。

### 1.4 小标题风格

小标题本身就带情绪或判断，不是中性标签。
- ✅ "都是嗨购害我！""福报跨越国界""流量，可以是自来水，也可能是泥石流"
- ❌ "第一章 背景介绍""当前市场分析""结论与展望"

**小标题要跟整体文风统一**。如果全文是锐利幽默的调性，小标题也应该是锐利幽默的，不能突然冒出一个故作深沉/故作神秘的标题。宁可不用小标题，也不要用一个格格不入的小标题。

## 二、语感节奏：口语化但有文人底子

### 2.1 句式特征

**① 长句不怕长，节奏靠标点断**

长句用逗号断成三四个小节奏，语义完整，读起来是一口气说完中间换气的感觉。

**② 短句爆破力来自"骂街感"**

短段的力量不是来自修辞，是来自直接的情绪表达。没有比喻，没有修辞，就是直给。
范例："人麻了。""被骂出翔了。""气死人。"

**③ 插入语和旁白感**

在句子中间插入个人评论或补充，制造"旁白感"，像跟读者私下聊天随口补一句。
范例："不加之一可能违反广告法，但我实在是找不到……"

### 2.2 用词偏好

**高频口语词汇**：坦白讲、有一说一、说白了、大白话就是、吭哧吭哧、磨磨蹭蹭
**语气词**：嘛、呢、啊、吧（自然穿插，不刻意）

**绝不出现的书面化词汇**：
- 不说"综上所述""由此可见""诚然""固然""无疑"
- 不说"笔者认为"，直接说"我觉得""我的判断是"
- 不用"值得注意的是""不得不说""众所周知"

### 2.3 文人底子（关键特色）

在大段口语中偶尔蹦出文言或半文言句式，制造"痞子文人"反差感。**不是通篇文绉绉，是突然来一句有底蕴的话。**

范例：
- "里面光怪陆离之处，不乏蝇营狗苟之辈。"
- "万般皆是命，半点不由人。"
- "破山中贼易，破心中贼难。"
- "此事在iOS灵动岛中亦有记载"（模仿古文纪录体，兼具幽默）

使用频率：一篇文章中最多1-2处。多了就成掉书袋了。

### 2.4 人称和视角

几乎全部第一人称，频繁暴露自己的身份、立场和利益关系。效果是读者觉得你不是居高临下分析，而是以同等身份聊天。

范例："作为一名2023款汉EV老韭菜车主""我一个文科生看了都直挠头"

### 2.5 词汇不重复（铁律）

同一篇文章中，同一个语气词、口头禅、评价词只出现一次。用过就换。

- "说实话"用了一次，后面要换"坦白讲""有一说一""讲真"等
- "说白了"用了一次，后面要换"翻译翻译""大白话就是"等
- 形容词、动词同理——不要反复用同一个词形容不同的事物

**排除情况：首尾呼应**。如果刻意在结尾重复开头的句子或关键词，以制造情感呼应和主旨收束的效果，这不算重复，是修辞手法，可以且应该这么用。

## 三、论证方式：横向类比是杀手锏

### 3.1 核心论证手法

**① 横向类比（出现频率最高，每篇必有）**

类比来源几乎全是**中国互联网本土**的东西。选择标准：不是找最准确的，是找读者最有体感的。

范例：
- Cursor被降维打击 → 类比微信公众号创业公司被小程序取代
- 国产安卓抄iPhone → 类比一群年轻人为融入高端圈子纷纷整容成同一副面孔
- 美国科技裁员 → 类比中国互联网996

**② 数据举证（克制使用）**

一篇文章2-4个关键数据点就够。数据后面跟一句口语化评论，把数据"翻译"成读者能感受到的东西。不堆砌数据。

**③ 用数据/事实论证，不用修辞排比（铁律）**

任何"不是A，是B""不是X，不是Y，是Z"的结构，本质上都是在秀文采而不是在讲道理。改法：直接给出具体原因、数据或研究结论。

- ❌ "这就是ChatGPT真正的护城河。不是技术领先，不是品牌认知，是你心里那句'换了太麻烦'。"
- ✅ "这就是ChatGPT真正的护城河，因为研究发现，每多加一个操作步骤，用户的流失率将呈指数级上升。"

前者是修辞，后者是论据。读者读完前者会觉得"这句话挺工整"，读完后者会觉得"确实是这么回事"。

**④ 戏剧化要演不要说（铁律）**

不要告诉读者"这很魔幻""这很讽刺""这很荒诞"——这是解说员在做旁白。好的写法是让读者自己感受到魔幻/讽刺/荒诞。用梗、吐槽、夸张的情绪反应来"演"。

- ❌ "操作流程堪称魔幻。"（旁白式评论）
- ✅ "Gemini 你小子浓眉大眼的也学坏了啊！"（角色代入式吐槽）

- ❌ "你品品这个画面。你在用 ChatGPT 帮你逃离 ChatGPT。"（引导式解说）
- ✅ 用一段提示词诱导ChatGPT，导出所有你的数据记忆，再打包发给另一个AI，一出赛博空间里的夫目前犯，这画面感，刺激……（过程本身就是画面）

区别：解说员告诉观众"这个场面很精彩"，脱口秀演员直接让观众笑出来。我们要做脱口秀演员。

**⑤ 类比要推到极端**

不要停在"差不多就是这样"，要找到那个最极端、最有画面感的落点。

- ❌ "就像当年大家管所有搜索引擎都叫'百度一下'，后来该用谷歌的还是用谷歌。"
- ✅ "有些搜索引擎先发优势十足，广告语足够洗脑，却在后来沦落成了测试网络是否正常的站点。"

前者类比停在半路，后者一刀捅到底，"测试网络是否正常的站点"是每个中国网民都有体感的画面。

**⑥ 相似素材合并，不要分写（铁律）**

如果两个案例/公司/产品做了几乎相同的事情，必须合并到一起写，不能拆成两段分别描述。否则读者会觉得"把名字换一换就是同一段话"，信息密度为零。

- ❌ 先单独写Gemini一段，隔两段又单独写Claude一段，内容几乎雷同
- ✅ 把两者合并："Anthropic 3月2号先动手，Google 3月26号跟上，两家前后脚做了同一件事"——一段搞定

合并后腾出的篇幅用来做更有价值的论证或增加新的信息。

**⑦ 活人感不能靠堆语气词硬凹（铁律）**

"好家伙""这画面我能回味一整天""整一出XXX啊"——这些语气词/评论如果没有具体画面支撑，就是在假装有活人感。真正的活人感来自：
1. 具体的、有画面感的案例（如"以前很多中国软件卸载时弹出二次元萌妹眼泪汪汪求你不要卸载"）
2. 情绪的自然递进（从案例到情绪，不是从语气词到语气词）
3. 适度的粗口和本能反应（"他妈的""卧槽"——是情绪到了自然冒出来的，不是为了表演）

**⑧ 例子够用就行，不要贪多**

同一论点的横向类比/案例，2个打透比3个蜻蜓点水强。每个例子都应该带来新的信息或新的情绪维度，如果两个例子讲的是同一个道理、给的是同一种感受，砍掉一个。

**⑨ 数据/案例必须核实时效性（硬约束）**

写文章时引用的模型名称、版本号、benchmark数据必须是截至发稿日期的最新信息。过时的数据比没有数据更致命，因为它直接暴露"作者不懂行"。

- ❌ 2026年的文章里用Claude 3.5、Gemini 2.0对比GPT-4o（已经是半年前甚至一年前的旧型号）
- ✅ 用截至发稿日期最新的模型版本和公开benchmark数据

**⑩ 具体场景/细节**

用一个极其具体的、最好是亲历的场景来佐证论点。读者一看就知道不是编的。

### 3.2 论证结构（松散但有骨架）

**事件/现象 → 表面解读（别人怎么看）→ 深层归因（我怎么看）→ 横向类比（历史上/其他领域类似的事）→ 延伸思考（所以呢？）→ 开放式收束**

其中"横向类比"和"开放式收束"几乎从不缺席。

## 四、收尾方式

### 绝不做的事
- 不总结全文
- 不"综上所述"
- 不"回顾本文三个要点"
- **不上价值、不教人做事、不说教（铁律）**——结尾不需要"思考""启示""价值"这些词。登味儿是大忌。

### 收尾模式

**① 反问收束**："你说呢？""你是怎么考虑这个问题的？"
**② 感慨/祝福式**："愿大家都能保住自己的工作。Peace and Love！"
**③ 警示/预判式**："但愿他能稳住自己。""且看它们会不会像之前一样滑跪吧。"
**④ 幽默/暴论收束**：用一个意想不到的角度或幽默的暴论结尾，让读者笑着合上文章。清醒但不沉重，有态度但不说教。

## 五、情绪与态度

- **有立场有锋芒**，但不是愤青，是"想通了之后的通透"
- **自嘲是常态**：拉近距离+增强可信度。你自己都承认是韭菜了，批评才更有说服力
- **有立场但不偏执**：明确表态，同时主动暴露另一面的合理性。展示思考过程而非灌输结论
- **情绪有层次**：同一篇文章里有多种情绪切换。不是通篇愤怒或通篇冷静
- **中国互联网本土梗信手拈来**：翟欣欣、罗永浩、夫目前犯、嗨购……这些梗的密度和精准度是AI很难凭空生成的

## 六、AI味黑名单（硬约束，逐条检查）

### 6.1 句式黑名单

| 编号 | 禁止句式 | 替代方案 |
|------|----------|----------|
| ❌1 | 破折号高频使用（一篇5个以上） | 用逗号断句。12篇范文里破折号极少 |
| ❌2 | "不是A，而是B"对仗句式 | 直接说"其实是B"或"说白了就是B" |
| ❌3 | "X，但Y"式转折短段（如"荒诞，但有效"） | 用完整句子做转折，或直接吐槽 |
| ❌4 | 排比三连 | 不追求工整，偶尔有也要参差不齐 |
| ❌5 | "让我们……"祈使句 | 从不用 |
| ❌6 | "值得注意的是""不得不说" | 删掉，直接说事 |
| ❌7 | "毫无疑问""无疑" | "我觉得""我的判断是" |
| ❌8 | "事实上""实际上"高频 | 偶尔一次可以，不能高频 |
| ❌9 | "这意味着" | 删掉或改成口语化表达 |
| ❌10 | "换句话说" | "说白了""大白话就是" |

### 6.2 标点黑名单

| 编号 | 禁止用法 | 说明 |
|------|----------|------|
| ❌1 | 双引号泛滥 | 只在引述原话或品牌名/特定概念时用。不给普通名词加双引号做"免责声明" |
| ❌2 | 破折号代替逗号 | 正常人用逗号的地方不要用破折号 |
| ❌3 | 省略号滥用 | 只在真正"留白"或"欲言又止"时用 |

### 6.3 结构黑名单

| 编号 | 禁止结构 | 说明 |
|------|----------|------|
| ❌1 | 通篇短段（每段1-2句） | 长段40%才是主力 |
| ❌2 | 每段都以金句结尾 | 很多段落就是平铺直叙，金句只在关键位置 |
| ❌3 | 结尾总结全文 | 从不"回顾" |
| ❌4 | 开头"在当今社会/时代" | 从不用空泛开头 |
| ❌5 | 编号列表做主要论证 | 偶尔可以，但不是主要论证方式 |

### 6.4 比喻黑名单

| 编号 | 禁止比喻方式 | 说明 |
|------|-------------|------|
| ❌1 | "就像XX一样"高频 | 类比方式更灵活，可以直接叙述另一个场景 |
| ❌2 | 过度精巧的比喻 | 比喻都要直白，不做精致小巧的文学修辞 |
| ❌3 | 故作深沉的隐喻堆砌 | "护城河""生态"这些词可以用，但不堆砌 |

## 七、段落节奏模板（可直接套用）

### 模板A：观点输出型（最常用）

```
[长段·80-150字] 铺陈背景/数据/事实
[中段·30-60字] 个人评价或吐槽
[长段·80-150字] 深入分析/论证/类比
[短段·10-30字] 情绪爆破或金句
[中段·40-80字] 延伸思考或转入下一话题
```

### 模板B：故事叙述型

```
[长段·100-200字] 讲一个完整的场景/经历
[短段·10-20字] 情绪反应
[中段·40-80字] 从故事中提炼观点
```

### 模板C：数据论证型

```
[中段·40-60字] 引出数据
[长段·80-120字] 展开数据细节+横向对比
[中段·30-50字] 用口语翻译数据的含义
[短段·10-20字] 吐槽或评价
```

## 八、自检流程（生成完正文后必须执行）

### 8.1 段落配比检查（最重要）
- [ ] 统计全文长段/中段/短段比例。长段是否≥35%？
- [ ] 是否存在连续3个以上短段（碎段问题）？
- [ ] 是否存在连续3个以上长段（闷段问题）？
- [ ] 整体节奏是否呈"呼吸"感（长-中-短交替）？

### 8.2 AI味逐项扫描（硬约束）
- [ ] 全文破折号数量≤2个？
- [ ] 全文双引号是否只用于引述原话/品牌名？
- [ ] 是否存在"不是A，而是B"或其变体（"不是X不是Y是Z"）？（找到即改为数据/事实论证）
- [ ] 是否存在"X，但Y"式转折短段？（找到即改）
- [ ] 是否存在排比三连？（找到即拆散）
- [ ] 是否存在"让我们/值得注意的是/不得不说/这意味着"等AI味过渡词？
- [ ] 是否存在逻辑连接词（首先/其次/然而/因此/此外/同时）？
- [ ] 结尾是否在总结全文？
- [ ] 是否存在"堪称XX""你品品"等旁白式评论？（改为角色代入式吐槽/情绪反应）
- [ ] 是否存在堆语气词硬凹活人感的段落？（无画面支撑的评论）
- [ ] 相似素材（多个公司/产品做了几乎相同的事）是否合并到一起写了？
- [ ] 数据/模型版本是否为截至发稿日期的最新信息？

### 8.3 横向类比检查
- [ ] 全文是否有≥2个横向类比？
- [ ] 类比来源是否包含中国互联网本土案例？
- [ ] 类比是否过于精巧/文学化？（类比都要直白）
- [ ] 类比是否推到了极端落点？（不能停在"差不多就是这样"）
- [ ] 例子数量是否克制？（同一论点2个打透即可，不贪多）

### 8.4 标题检查
- [ ] 读者看完标题会不会想"然后呢"？
- [ ] 标题有没有把事情说全？（说全了就失败了）
- [ ] 标题里每个字是否都真实？

### 8.5 开头检查
- [ ] 前3段信息密度是否足够？（梗/案例/数据/反转是否密集释放）
- [ ] 是否存在干巴巴的设问开头？（如果是，砍掉重写）

### 8.6 情绪层次检查
- [ ] 全文是否只有一种情绪？（应有多种情绪自然切换）
- [ ] 是否有自嘲元素？（增强亲近感和可信度）
- [ ] 立场是否明确但不偏执？
- [ ] 戏剧化场景是"演"出来的还是"说"出来的？（应该演不应该说）
- [ ] 小标题是否跟整体文风统一？（格格不入的砍掉或改掉）
- [ ] 结尾是否有登味/说教/上价值？（找到即改为幽默/暴论/轻松收束）

### 8.7 词汇多样性检查
- [ ] 同一个语气词/口头禅是否出现了多次？（找到即替换）
- [ ] 首尾呼应除外——刻意的重复是修辞手法，不算重复
- [ ] 是反问/感慨/警示收束，而非总结？
- [ ] 有没有留白或引发思考的空间？

### 8.8 整体检查
- [ ] 通读一遍，有没有哪里想跳过？（有就改）
- [ ] 情绪线是否连贯自然？
- [ ] 字数是否适合目标平台？
